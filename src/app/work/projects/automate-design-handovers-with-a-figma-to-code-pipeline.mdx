## Overview

This project organizes portfolio work as editable MDX content so each project can have metadata, images, summaries, and long-form notes without changing the application code.

## Key Features

- **MDX project pages**: Each project can include frontmatter, headings, lists, and rich body content.
- **Reusable metadata**: Titles, dates, summaries, links, and team details are read consistently across the site.
- **Scalable structure**: New projects can be added by creating a focused content file.
- **Clear presentation**: The work page and project detail pages stay aligned through shared rendering logic.

## Technologies Used

- **MDX** for content authoring.
- **Next.js** for routing and page generation.
- **React** for rendering project components.
- **TypeScript** for shared types and predictable data.

## Challenges and Learnings

The biggest lesson was keeping content flexible without making the editing experience complicated. A simple frontmatter structure keeps project details consistent while still allowing each page to tell its own story.

## Outcome

The portfolio now has a project system Chan can continue using as new work is completed and published.
